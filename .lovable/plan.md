
# เอกสารโครงสร้างแอป BOB (Bud Over Bills)

## 📱 ภาพรวมแอปพลิเคชัน

**ชื่อแอป:** BOB (Bud Over Bills)  
**วัตถุประสงค์:** แอปจัดการข้อตกลงยืม-ให้ยืมเงินระหว่างเพื่อน พร้อมระบบติดตามการชำระเงิน  
**เทคโนโลยี:** React + Vite + TypeScript + Tailwind CSS + Supabase (Lovable Cloud)

---

## 🏠 โครงสร้างหน้าหลัก (Routes)

### 1. หน้าแรก (Index) - Route: `/`
**หน้าที่:** Dashboard แสดงภาพรวมการยืม-ให้ยืมทั้งหมด

**ส่วนประกอบ:**
- **Header:** แสดงชื่อผู้ใช้ + ปุ่มค้นหา + ปุ่มแจ้งเตือน
- **DashboardStats:** สรุปยอดเงินที่ให้ยืม/ยืม
- **PendingActionsCard:** รายการที่ต้องดำเนินการ (ยืนยันสลิป, จ่ายงวด ฯลฯ)
- **PendingAgreements:** ข้อตกลงที่รอยืนยัน
- **PaymentCalendar:** ปฏิทินแสดงวันครบกำหนดชำระ (แยกสีตาม role: เขียว=ผู้ให้ยืม, ส้ม=ผู้ยืม)
- **DebtCard (รายการ):** แสดงข้อตกลงที่ active พร้อมสถานะ

**ปุ่มที่กดได้:**
- กดที่ DebtCard → ไปหน้ารายละเอียดข้อตกลง
- กดวันในปฏิทิน → แสดงรายการนัดวันนั้น
- สลับ Tab ผู้ให้ยืม/ผู้ยืม → filter ข้อตกลง

---

### 2. สร้างข้อตกลง (CreateAgreement) - Route: `/create`
**หน้าที่:** สร้างข้อตกลงยืมเงินใหม่

**Flow:**
1. เลือกคู่สัญญาจากรายชื่อเพื่อน (บังคับ)
2. กรอกจำนวนเงิน
3. เลือกจำนวนงวด (1-48 งวด)
4. เลือกความถี่ (รายวัน/รายสัปดาห์/รายเดือน)
5. เลือกประเภทดอกเบี้ย (ไม่มี/Flat Rate/Effective Rate)
6. กรอกบัญชีธนาคารสำหรับรับโอน
7. ยืนยันด้วยรหัสผ่าน → ส่งข้อตกลงไปยังคู่สัญญา

**ปุ่มที่กดได้:**
- เลือกเพื่อน → เปิด Dialog รายชื่อเพื่อน
- เลือกธนาคาร → Dropdown รายชื่อธนาคารไทย
- ปรับ Slider จำนวนงวด
- สลับ Tab ประเภทดอกเบี้ย
- ปุ่ม "สร้างข้อตกลง" → เปิด Password Dialog

**กฎสำคัญ:**
- ผู้สร้าง = ผู้ให้ยืม (Lender)
- ต้องเลือกเพื่อนจากระบบเท่านั้น (ไม่ใช่กรอกชื่อเอง)
- มี Quota ฟรี 2 ครั้ง/เดือน → จากนั้นต้องซื้อ credit

---

### 3. ยืนยันข้อตกลง (AgreementConfirm) - Route: `/agreement/:id/confirm`
**หน้าที่:** ให้คู่สัญญายืนยันรับ/ปฏิเสธข้อตกลง

**Flow สำหรับผู้ให้ยืม (Lender):**
1. อัปโหลดสลิปโอนเงินให้ยืม (บังคับ)
2. ตรวจสอบรายละเอียดข้อตกลง
3. อ่านและยอมรับข้อความทางกฎหมาย
4. ยืนยันด้วยรหัสผ่าน

**Flow สำหรับผู้ยืม (Borrower):**
1. ดูสลิปโอนเงินจากผู้ให้ยืม
2. ตรวจสอบรายละเอียดข้อตกลง
3. ยืนยันว่าได้รับเงินแล้ว + ยอมรับข้อตกลง
4. ยืนยันด้วยรหัสผ่าน

**ปุ่มที่กดได้:**
- อัปโหลดสลิป → เลือกไฟล์ JPG/PNG/PDF
- ดูสลิป → เปิด Preview
- ยืนยันข้อตกลง → Password Dialog
- ปฏิเสธข้อตกลง → ยกเลิกข้อตกลง

**กฎสำคัญ:**
- ต้องมีทั้งสองฝ่ายยืนยัน → สถานะเปลี่ยนเป็น "active"
- บันทึก IP + Device ไว้เป็นหลักฐานทางกฎหมาย
- มีข้อความยืนยันเป็นภาษาไทยตาม พ.ร.บ.ธุรกรรมอิเล็กทรอนิกส์

---

### 4. รายละเอียดข้อตกลง (DebtDetail) - Route: `/debt/:id`
**หน้าที่:** แสดงรายละเอียดข้อตกลงและจัดการงวดชำระ

**ส่วนประกอบ:**
- ข้อมูลคู่สัญญา + สถานะ
- ยอดคงเหลือ / ยอดจ่ายแล้ว
- Progress bar การชำระ
- รายการงวดทั้งหมด (แยกสถานะ: pending/paid/overdue)
- ข้อมูลบัญชีธนาคารสำหรับโอน
- ประวัติสลิปโอน

**ปุ่มที่กดได้ (สำหรับผู้ยืม):**
- กดงวดที่ต้องจ่าย → เปิด PaymentDialog
- อัปโหลดสลิป → แจ้งชำระ
- ขอเลื่อนกำหนด → เปิด RescheduleRequestDialog

**ปุ่มที่กดได้ (สำหรับผู้ให้ยืม):**
- กดงวดที่รอยืนยัน → ดูสลิป + ยืนยัน/ปฏิเสธ
- ดาวน์โหลด PDF → export ข้อตกลงเป็น PDF

**สถานะงวด:**
- `upcoming` = ยังไม่ถึงกำหนด
- `pending` = อัปโหลดสลิปแล้วรอยืนยัน
- `paid` = จ่ายแล้ว
- `overdue` = เลยกำหนด
- `rejected` = สลิปถูกปฏิเสธ

---

### 5. เพื่อน (Friends) - Route: `/friends`
**หน้าที่:** จัดการรายชื่อเพื่อนสำหรับสร้างข้อตกลง

**Flow เพิ่มเพื่อน:**
1. กดปุ่ม "+" → เปิด Dialog เพิ่มเพื่อน
2. กรอกชื่อ + เบอร์โทร (optional)
3. กดบันทึก → เพิ่มในรายชื่อ

**ปุ่มที่กดได้:**
- ค้นหาเพื่อน
- เพิ่มเพื่อนใหม่
- แก้ไขชื่อ/เบอร์
- ลบเพื่อน

**หมายเหตุ:** เพื่อนที่มี friend_user_id (ลิงก์กับบัญชีจริง) สามารถแชทตรงได้

---

### 6. แชท (Chat) - Route: `/chat` และ `/chat/:chatId`
**หน้าที่:** ระบบสนทนาระหว่างคู่สัญญา

**ประเภทแชท:**
1. **Agreement Chat** - แชทในข้อตกลง (เห็นเฉพาะคู่สัญญา)
2. **Direct Chat** - แชทตรงกับเพื่อน

**ส่วนประกอบ:**
- **ChatThreadList:** รายการห้องแชททั้งหมด (เรียงตาม last message)
- **ChatRoom:** ห้องแชทหลัก
- **ChatMessageBubble:** ฟองข้อความ (ซ้าย=อีกฝ่าย, ขวา=ตัวเอง)

**ปุ่มที่กดได้:**
- เลือกห้องแชท → เปิดห้อง
- Tab "เพื่อน" → ดูรายชื่อเพื่อนและเริ่มแชทใหม่
- ส่งข้อความ
- กลับไปรายการแชท

**Layout แบบ LINE:**
- ข้อความของตัวเอง → ชิดขวา + สีฟ้า
- ข้อความอีกฝ่าย → ชิดซ้าย + สีเทา

---

### 7. แจ้งเตือน (Notifications) - Route: `/notifications`
**หน้าที่:** แสดงการแจ้งเตือนทั้งหมด

**ประเภทแจ้งเตือน:**
- `payment_reminder` - เตือนก่อนครบกำหนด 💰
- `payment_received` - ได้รับการชำระ ✅
- `agreement_completed` - ข้อตกลงเสร็จสิ้น 🎉
- `friend_request` - คำขอเป็นเพื่อน 👋
- `security_alert` - แจ้งเตือนความปลอดภัย 🔒

**ปุ่มที่กดได้:**
- กดแจ้งเตือน → ไปยังหน้าที่เกี่ยวข้อง
- อ่านทั้งหมด
- ลบแจ้งเตือน

---

### 8. โปรไฟล์ (Profile) - Route: `/profile`
**หน้าที่:** แสดงข้อมูลผู้ใช้และการตั้งค่า

**ส่วนประกอบ:**
- รูป Avatar + ชื่อ + อีเมล
- ส่วนเพิ่มเพื่อนด้วย QR Code
- คำขอเป็นเพื่อนที่รอดำเนินการ
- ข้อมูลบัญชีธนาคาร
- Badge ความสำเร็จ (จ่ายตรงเวลา/ปิดหนี้/สมาชิกใหม่)
- สถิติข้อตกลงทั้งหมด
- เมนูต่างๆ

**ปุ่มที่กดได้:**
- อัปโหลด Avatar
- แก้ไขชื่อ
- ไปหน้าเพื่อน
- ไปหน้า Badges
- ไปหน้าประวัติ
- ไปหน้าตั้งค่า
- ไปหน้า Subscription
- Admin Hub (เฉพาะ Admin/Mod)
- ออกจากระบบ

---

### 9. ตั้งค่า (Settings) - Route: `/settings`
**หน้าที่:** ปรับแต่งการใช้งานแอป

**หมวดการตั้งค่า:**
1. **ภาษา:** ไทย/อังกฤษ/จีน/ญี่ปุ่น/เกาหลี
2. **แจ้งเตือน:**
   - Push Notification
   - อีเมล
   - เตือนก่อนครบกำหนด
   - อัปเดตข้อตกลง
3. **ความเป็นส่วนตัว:**
   - แสดงโปรไฟล์ต่อสาธารณะ
   - แสดงกิจกรรม
4. **ธีม:**
   - โหมดมืด/สว่าง
5. **ข้อมูลกฎหมาย:**
   - ข้อกำหนดการใช้งาน
   - นโยบายความเป็นส่วนตัว
   - PDPA

### 10. Subscription - Route: `/subscription`
**หน้าที่:** ซื้อ Credit สำหรับสร้างข้อตกลง (แบบ "เลี้ยงกาแฟ")

**Tier กาแฟ:**
| ระดับ | ราคา | Credit ได้ |
|-------|------|-----------|
| กาแฟรถเข็น 🍵 | ฿19 | 1 สิทธิ์ |
| กาแฟหรู ☕ | ฿29 | 2 สิทธิ์ |
| กาแฟพรีเมี่ยม ✨ | ฿49 | 4 สิทธิ์ |

**ระบบ:**
- ฟรี 2 ครั้ง/เดือน
- หมดฟรี → ซื้อ Credit
- Credit ไม่หมดอายุ

---

### 11. ประวัติ (History) - Route: `/history`
**หน้าที่:** แสดงประวัติข้อตกลงทั้งหมด

**Tab:**
1. **ทั้งหมด:** ข้อตกลงทุกสถานะ
2. **ปิดสำเร็จ:** เฉพาะที่ completed

**ข้อมูลสรุป:**
- จำนวนข้อตกลงที่ปิด
- ยอดเงินต้นรวม
- ยอดดอกเบี้ยรวม
- จำนวนที่เป็นผู้ให้ยืม/ผู้ยืม

---

### 12. Auth - Route: `/auth`
**หน้าที่:** เข้าสู่ระบบ/สมัครสมาชิก

**วิธีเข้าสู่ระบบ:**
1. **Google OAuth** - เข้าสู่ระบบด้วย Google
2. **Email/Password** - สมัครด้วยอีเมล

**Flow สมัครสมาชิก:**
1. กรอกอีเมล + รหัสผ่าน
2. รอ Email Verification
3. กรอกข้อมูลส่วนตัว (PersonalInfoOnboarding)
4. ยอมรับ PDPA (PDPAConsent)
5. เข้าใช้งานได้

**Flow Google:**
1. กด "ดำเนินการด้วย Google"
2. เลือกบัญชี Google
3. กรอกข้อมูลส่วนตัว + PDPA
4. เข้าใช้งานได้

## 🔐 ระบบ Admin

### Admin Hub - Route: `/admin`
**เข้าถึงได้:** Admin และ Moderator เท่านั้น

**หน้าย่อย:**
1. **จัดการคอนเทนท์** (`/admin/feed`) - สร้าง/แก้ไขบทความ
2. **Security Dashboard** (`/admin/security`) - ดู Activity Logs
3. **จัดการสิทธิ์ผู้ใช้** (`/admin/users`) - แต่งตั้ง/ถอดสิทธิ์ Admin/Mod

**การยืนยันตัวตน:**
1. Admin กดเข้า Admin Hub
2. ระบบส่ง OTP ไปยังอีเมล
3. กรอก OTP → เข้าถึงได้

## 🗄️ โครงสร้างข้อมูลหลัก

### Debt Agreement (ข้อตกลง)
```
- id: UUID
- lender_id: ผู้ให้ยืม
- borrower_id: ผู้ยืม
- borrower_name: ชื่อผู้ยืม
- principal_amount: เงินต้น
- interest_rate: ดอกเบี้ย (%)
- interest_type: none/flat/effective
- total_amount: ยอดรวม
- num_installments: จำนวนงวด
- frequency: daily/weekly/monthly
- start_date: วันเริ่มต้น
- status: pending_confirmation/active/completed/cancelled
- lender_confirmed: ผู้ให้ยืมยืนยันแล้ว
- borrower_confirmed: ผู้ยืมยืนยันแล้ว
- transfer_slip_url: สลิปโอนเงินให้ยืม
- bank_name/account_number/account_name: บัญชีรับเงิน
```

### Installment (งวดชำระ)
```
- id: UUID
- agreement_id: ข้อตกลงที่เกี่ยวข้อง
- installment_number: งวดที่
- due_date: วันครบกำหนด
- amount: ยอดเงิน
- principal_portion: ส่วนเงินต้น
- interest_portion: ส่วนดอกเบี้ย
- status: pending/paid
- paid_at: วันที่จ่าย
- payment_proof_url: สลิปการจ่าย
```

### Message (ข้อความ)
```
- id: UUID
- agreement_id / direct_chat_id
- sender_id: ผู้ส่ง
- content: เนื้อหา
- created_at: เวลาส่ง
- read_at: เวลาอ่าน
```

---

## 🔒 กฎความปลอดภัย (RLS)

1. **ข้อตกลง:** เห็นได้เฉพาะ lender_id หรือ borrower_id = current user
2. **ข้อความ:** เห็นได้เฉพาะผู้ที่อยู่ในห้องแชท
3. **เพื่อน:** เห็นได้เฉพาะ user_id = current user
4. **สลิป:** เก็บใน Private Storage bucket
